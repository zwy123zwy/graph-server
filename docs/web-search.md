# 网络搜索与抓取能力说明

本后端提供两类与「网络」相关的能力，可与大模型配合实现「搜索 → 选链接 → 抓取正文 → 总结」的流程。

## 一、能力概览

| 能力 | 工具名 | 作用 |
|------|--------|------|
| 网页搜索 | `web_search` | 根据关键词在互联网上搜索，返回摘要/相关链接/维基等 |
| 抓取网页正文 | `crawl_url` | 对指定 URL 发起请求，将 HTML 转为纯文本返回，供模型总结 |

流程示例：用户问「最近 LangGraph 有什么更新」→ 模型调用 `web_search(query="LangGraph 更新")` → 拿到若干链接与摘要 → 若需某条链接的详细内容，再调用 `crawl_url(url="...")` → 根据正文生成回答。

## 二、实现位置

- **搜索**：`src/tools/web-search.ts`（DuckDuckGo + Wikipedia 为默认；可选模力方舟/快搜 API）
- **抓取**：`src/tools/crawl-url.ts`（fetch + 简单 HTML 转文本）
- **接入 Agent**：`src/agent.ts` 中注册上述工具，并在系统提示中约定何时使用

## 三、人为判断点（需你根据业务决定的部分）

以下配置或策略没有唯一「正确」答案，需要你按合规、成本、体验自行取舍。

### 1. 搜索 API 与密钥

- **是否使用付费/额度制 API**：当前默认用 DuckDuckGo Instant Answer + Wikipedia（无需密钥）。若在 `.env` 中配置 `MOARK_API_TOKEN` 或 `KUAISOU_API_KEY`，会优先使用对应 API 的网页结果（标题/链接/摘要更完整）。
- **人为判断**：是否注册模力方舟/快搜、是否接受其条款与额度限制、生产环境是否改用其他搜索 API。

### 2. 搜索结果数量

- **位置**：`src/tools/web-search.ts` 中的 `MAX_WEB_RESULTS`（当前为 8）。
- **人为判断**：条数越多，上下文越长、延迟与 token 消耗越大；可根据模型上下文与产品需求调整。

### 3. 何时触发搜索（是否调用 web_search）

- **位置**：`src/agent.ts` 中的 `shouldUseTools()`，通过关键词（如「搜索」「最新」「实时」「新闻」等）和简单规则判断「是否需要工具」；若不需要则可能阻止模型调用工具。
- **人为判断**：当前规则是启发式的，可能误判。你可以放宽（更多场景允许搜索）或收紧（减少不必要的搜索与成本）。

### 4. 抓取目标：允许的 URL 范围

- **位置**：`src/tools/crawl-url.ts` 目前仅校验「是否为 http/https」，未做域名白名单/黑名单。
- **人为判断**：是否只允许抓取公网、是否排除内网或敏感域名、是否禁止某些国家/地区站点，需按合规与安全策略在代码中增加校验（如 `parsed.hostname` 白名单或黑名单）。

### 5. 抓取超时与正文长度

- **位置**：`src/tools/crawl-url.ts` 中 `FETCH_TIMEOUT_MS`（当前 15 秒）、`MAX_TEXT_LENGTH`（当前 18_000 字符）。
- **人为判断**：超时过长会拖慢对话，过短容易抓取失败；正文过长会占满上下文、过短可能信息不足。需根据模型上下文与典型页面大小调整。

### 6. HTML 转文本质量

- **位置**：`src/tools/crawl-url.ts` 中的 `stripHtmlToText()`，当前为简单实现（去 script/style、块级换行、去标签、基础实体解码）。
- **人为判断**：若需要更好排版或更精准的正文抽取，可引入 `html-to-text` 等库，或对接专门的内容提取服务，并在该工具内替换实现。

### 7. 内容安全与合规

- 当前未对搜索关键词、抓取 URL 或返回正文做内容过滤。
- **人为判断**：是否需要对查询/URL/正文做敏感词过滤、违法内容拦截、版权与引用标注等，需在业务层或工具层增加策略。

### 8. 限流与防护

- 当前未对 `web_search` / `crawl_url` 做按用户或按 IP 的限流。
- **人为判断**：是否加 QPS/每日调用上限、是否对异常请求（如大量爬取）做限制，以避免滥用和第三方 API 封禁。

---

以上为与「网络搜索能力」相关的主要人为判断点；具体实现与数值需你根据运行环境和产品要求决定。
